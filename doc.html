REGISTER / LOG IN

Une fois l'application lancée, il vous est possible de vous enregistrer en tant que nouvel utilisateur ou bien de vous login en tant qu'administrateur avec les identifiants par défaut. Partons du principe que vous êtes un tout nouvel utilisateur : voici le menu d'enregistrement de compte que vous aurez après avoir appuyé sur le bouton "create an Account" en bas de l'écran. Si vous souhaitez vous login :

  • Enter mail : Entrez votre adresse mail d'inscription
  • Enter password : Entrez votre mot de passe choisi à l'inscription
  • Validate : Cliquez pour vous connecter.
  • "Remember my account" est une case que vous pouvez cocher pour conserver votre mail et mot de passe à l'écran de connection. Ne recochez cete case que si vous souhaitez conserver un autre mail / mot de passe.



  • Enter your name: insérez votre nom de famille.
  • Enter your firstname: insérer votre prénom.
  • Enter your mail: insérez votre adresse mail.
  • Enter your password: insérez votre mot de passe. Celui-ci doit faire au moins 12 caractères, comporter au moins 3 chiffres, 3 minuscules, 2 majuscules et 2 caractères spéciaux.
  • Choose your role: sélectionnez le rôle que vous souhaitez obtenir, "Architect" pour architecte et "Customer" pour client. Ce sera à l'administateur de vous valider manuellement.

    • Pour finir, appuyez sur "Validate" pour confirmer votre inscription. Vous pourrez dès à présent vous connecter.

    UPDATE INFO

    Il vous est possible de changer vos informations de sécurité à n'importe quel moment. Pour cela, une fois connecté, accédez à cet écran comme suit: "Account" -> "Update Infos".

    Vous pourrez donc y changer votre adresse mail et/ou votre mot de passe.

  • New Mail Adress: entrez votre nouvelle adresse mail ou laisser vide pour la laisser inchangée.
  • [OBLIGATOIRE] Password: Si vous souhaitez changer un de vos paramètres, vous devrez entrer votre mot de passe actuel.
  • New password: Nouveau mot de passe qui doit respecter la syntaxe suivante: comporter au moins 3 chiffres, 3 minuscules, 2 majuscules et 2 caractères. Laisser vide pour le laisser tel quel.
  • Validate : confirmez votre changeemnt de mail / mot de passe. Les critères cités plus haut doient être valides.

    • FILTER

    Une fois validé dans un groupe ou en tant qu'administarteur, il vous est possible de filtrer les résultats de vos recherches, pour filtrer les noms d'utilisateurs, les prénoms ou le nom de projet. Tous les filtres peuvent être appliqué en même temps.

  • Les filtres textuels filtrent les résultats, n'affichant le contenu que si celui-ci contient et commence par les mêmes caractères quevous avez entré.
  • Il vous faudra raffraichir le filtrage en appuyant sur un bouton de confirmation, afin de pouvoir lancer le filtre.

    • CONVERSATIONS

    La messagerie est notre partie phare de l'application ! En tant que client, vous pourrez communiquer avec votre architecte référrent pour chaqun de vos projet, en plus de pouvoir discuter avec les autres architectes qui auront prêté main forte à votre référrent. Une fonction de filtre y est présente, à savoir:

  • Le nom de Projet
  • le nom d'utilisateur
  • Le prénom d'utilisateur
    • Les architectes peuvent faire de même.

    En haut se situent les projets et les émeteurs des messages trié du plus réçent au plus ancien.

  • Cliquez dessus pour afficher tous les messages entre lui et vous dans l'interface en bas.
  • Cliquez sur "Answer" pour répondre au message.

    • MODIFY PROJECT

    En tant qu'architecte, vous avez la capacité de modifier le contenu de n'importe quel projet sur lequel vous êtes référent. Pour modifier une donnée, il vous suffit de double-clicker dessus puis de taper le texte que vous souhaitez. Une fois cela fait, appuyez sur la touche "ENTRER" de votre clavier et hop, l'information est mise à jour !

    CRUD
  • Users

    • L'interface de gestion d'utilisateur permet à l'administrateur de gérer tous les utilistauers qui ne sont pas administrateurs. Il est possible de filtrer les utilisateurs selon leur rôle: "architect" ou "customer", correspondant à "architecte" et "client".

  • name
    • Le nom de l'utilisateur.
  • firstname
    • Le prénom de l'utilisateur.
  • mail
    • Il s'agit de l'adresse mail de l'utilisateur pour se connecter à l'application. Nous vous conseillons de n'y toucher que si l'utilisateur perd son adresse mail.
  • password
    • Il s'agit du mot de passe en clair de l'utilisateur pour se connecter à l'application. Nous vous conseillons de n'y toucher que si l'utilisateur perd son accès.
  • role
    • Permet de modifier le role de l'utilisateur et par conséquent ses permissions. Si vous vous trompez d'orthographe lors du filtre, retirez le filtre pour retrouver l'utilisateur en question.
  • signdatetime
    • La date et l'heure d'inscription de l'utilisateur sur l'application. Doit respecter la syntaxe suivante: AAAA-MM-JJ HH:mm.

    Vous pouvez sélectionner un utilisateur puis cliquer sur "Delete an user" pour l'effacer.

    En cliquant sur "Adding an user", vous accédez à un écran d'ajout d'utilisateur, afin de pré-créer ce dernier. Dans l'ordre, vous pourrez y insérer les informations suivantes:

  • Le nom
  • Le prénom
  • L'adresse mail
    • Doit être unique, sinon, l'action est refusée.
  • le mot de passe
    • Doit faire 12 caractères minimum et doit contenit au moins 3 minuscules, 3 chiffres, 2 majuscules et 2 caractères spéciaux. Sinon, l'action est refusée
  • Le role
    • Le menu déroulant vous propose soit Customer (client), soit architecte. Cliquez sur "VALIDATE" pour inscrire l'utilisateur. Sa date d'inscription est comprise automatiquement.

    Lorsqu'un utilisateur s'inscrit sur l'application, celui-ci n'a pas automatiquement les droits autre que modifier son mail et/ou mot de passe. C'est donc à vous de valider son compte ou de le refuser. En cas de refus, l'utilisateur est supprimé.
    Pour valider ou refuser un utilisateur, cliquez sur sa ligne, puis cliquez sur "Accet selected user registration" ou "Reject selected user registration".

  • role
    • présenté avec un "*" devant le rôle, cela signifie le rôle que l'utilisateur souhaite obtenir lors de son inscription. Ce rôle lui sera donné en cas de requête acceptée de votre part.
